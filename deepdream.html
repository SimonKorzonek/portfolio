DEEPDREAM

This project was mostly focused on image processing with Tensorflow and Pillow. Program "dreamyfies" given source image. It's very hard to set it up correctly for getting effect you want. Also, it's easy to run out of RAM memory (could be adressed with randomly cutting image for every octave and dreamifying each cutted tile separately)

×

Object recognition alogorythm guesses what it sees on picture (calculating gradient of a dream), then amplifying it and mixing with raw image over and over again. This leads to creation of psychodelic, recurrent patterns within image.

Used technologies

- InceptionV3 model made by Google, with weights pre-trained on Imagenet

- Keras over Tensorflow 2.3 backend

- Pillow image processing

Functionalities

"Deepdream" has many parameters you can tweak. Here you can learn how each of them affects the dream:

choosen_layers - layers of Inception model used in object recognition process. Lower layers will see basic patterns, higher ones will produce more complex patterns made of sum of simpler ones. You have pick at least two of them. Note that higher layers are computationaly heavier and more RAM consuming.

octaves - number of iterations of gradient ascent applied on different scales (octave_scale). It allowes patterns generated at smaller scales to be incorporated into patterns at higher scales and filled with additional detail.

octave_scale - ratio of image scaling with each octave.

step_size/steps - higher values will make dream more intense.

target_size - size of image before upscaling. Size of image after processing will be equal to (target_size * octave_scale) * octaces.
